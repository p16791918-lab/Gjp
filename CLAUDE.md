# Claude Code 작업 플로우

교시별 분석은 반드시 아래 섹션 순서대로 진행하며, **각 섹션 완료 후 반드시 멈추고 사용자의 확인을 받은 뒤** 다음 섹션으로 넘어간다. 여러 교시를 한 번에 처리하지 않는다.

---

## 교시별 분석 플로우 (1교시 → 2교시 → 3교시 순서로 각각 진행)

### 섹션 A: 지침 읽기

**작업 내용:**
- 해당 교시의 정답지(XLSX) 파일을 읽어 과목 구성과 문항 수를 파악한다
- 기존 `make_pdf_*.py` 스크립트가 있다면 현재 구조를 확인한다
- 분석 대상 연도(2021, 2023, 2024, 2025)의 파일 목록을 확인한다

**완료 기준:** 해당 교시 파일 구조 및 과목 구성 파악 완료

> **멈춤 포인트:** 섹션 A 완료. 파악한 파일 목록과 과목 구성을 보고하고, 섹션 B로 넘어갈지 확인을 받는다.

---

### 섹션 B: 문제 읽기

**작업 내용:**
- 해당 교시의 각 연도별 문제 PDF를 순서대로 읽는다 (한 번에 한 연도씩)
- 각 문항의 주제, 핵심 키워드, 출제 패턴을 기록한다
- 연도 간 반복 출제 항목을 표시한다

**완료 기준:** 4개 연도 문제 전체 검토 및 반복 출제 항목 목록 작성 완료

> **멈춤 포인트:** 섹션 B 완료. 연도별 문제 검토 결과와 반복 출제 항목 목록을 보고하고, 섹션 C로 넘어갈지 확인을 받는다.

---

### 섹션 C: 빈출 유형 분석

**작업 내용:**
- 3~4년 연속 출제: ★★★ 최빈출 유형으로 분류
- 2년 출제: ★★ 준빈출 유형으로 분류
- 각 유형별 핵심 포인트 및 오답 선택지 패턴을 정리한다
- 주제별 핵심 요약을 작성한다

**완료 기준:** 최빈출/준빈출 유형 목록 및 핵심 포인트 정리 완료

> **멈춤 포인트:** 섹션 C 완료. 분류된 빈출 유형 목록을 보고하고, 섹션 D(스크립트 작성)로 넘어갈지 확인을 받는다.

---

### 섹션 D: PDF 생성 스크립트 작성

**작업 내용:**
- 섹션 C의 분석 결과를 바탕으로 `make_pdf_*.py` 스크립트를 작성/수정한다
- 스크립트를 실행하여 PDF가 정상 생성되는지 확인한다
- 오류가 발생하면 수정 후 재실행한다

**완료 기준:** PDF 파일 정상 생성 확인

> **멈춤 포인트:** 섹션 D 완료. 생성된 PDF 경로와 내용을 보고하고, 커밋/다음 교시 진행 여부를 확인을 받는다.

---

## 규칙 요약

| 규칙 | 내용 |
|------|------|
| 교시 순서 | 1교시 → 2교시 → 3교시 (한 번에 하나씩) |
| 섹션 순서 | A → B → C → D (건너뛰지 않음) |
| 멈춤 조건 | 각 섹션 완료 후 반드시 사용자 확인 대기 |
| 파일 읽기 | 한 번에 한 연도 파일만 읽음 (컨텍스트 초과 방지) |
