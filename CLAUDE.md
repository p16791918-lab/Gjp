# Claude Code 작업 플로우

작업을 한 번에 처리하면 오류가 발생할 수 있으므로, 아래 섹션을 **순서대로** 처리하고 각 섹션 완료 후 반드시 멈춰서 사용자의 확인을 받아야 합니다.

---

## 섹션 1: 지침 읽기

**작업 내용:**
- 이 CLAUDE.md 파일을 읽고 프로젝트 구조와 규칙을 파악한다
- 프로젝트 디렉토리 구조를 확인한다 (`/home/user/Gjp/`)
- 기존 스크립트 파일 목록을 확인한다

**완료 기준:** 프로젝트 목적, 파일 구조, 스크립트 역할을 파악한 상태

> **멈춤 포인트:** 섹션 1 완료. 파악한 내용을 요약 보고하고, 섹션 2로 넘어갈지 확인을 받는다.

---

## 섹션 2: 이슈/문제 읽기

**작업 내용:**
- GitHub 이슈를 조회한다 (`mcp__github__list_issues` 또는 `mcp__github__issue_read`)
- 현재 작업 브랜치의 PR이 있다면 PR 내용을 읽는다
- 해결해야 할 문제 또는 구현해야 할 기능을 명확히 파악한다

**완료 기준:** 이슈/요구사항을 정확히 이해한 상태

> **멈춤 포인트:** 섹션 2 완료. 파악한 이슈/요구사항을 요약 보고하고, 섹션 3으로 넘어갈지 확인을 받는다.

---

## 섹션 3: 계획 수립

**작업 내용:**
- 이슈를 해결하기 위한 구체적인 작업 계획을 세운다
- 수정/생성할 파일 목록과 변경 내용을 나열한다
- 예상 위험 요소나 주의사항을 정리한다

**완료 기준:** 단계별 실행 계획 수립 완료

> **멈춤 포인트:** 섹션 3 완료. 작업 계획을 보고하고, 섹션 4(구현)로 넘어갈지 확인을 받는다.

---

## 섹션 4: 구현

**작업 내용:**
- 섹션 3의 계획에 따라 코드를 작성/수정한다
- 각 파일 변경 후 간단히 변경 내용을 보고한다
- 스크립트 실행이 필요하면 실행하고 결과를 보고한다

**완료 기준:** 모든 변경사항 구현 완료, 오류 없음

> **멈춤 포인트:** 섹션 4 완료. 구현 내용을 요약 보고하고, 섹션 5(검토 및 커밋)로 넘어갈지 확인을 받는다.

---

## 섹션 5: 검토 및 커밋

**작업 내용:**
- `git diff`로 변경사항을 최종 검토한다
- 문제없으면 커밋 메시지를 작성하고 커밋한다
- `git push -u origin <branch-name>`으로 원격에 푸시한다

**완료 기준:** 커밋 및 푸시 완료

> **멈춤 포인트:** 섹션 5 완료. 커밋 해시와 푸시 결과를 보고하고 작업 종료.
